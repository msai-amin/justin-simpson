# Justin Simpson — personal site

A personal homepage for Justin Simpson, philosopher of **environment & technology**.

The whole page lives in two moods that one control swaps between:

- **Physis** (φύσις, _the self-arising_) — warm parchment, serif type, flowing organic contours.
- **Technê** (τέχνη, _the bringing-forth_) — graphite, monospace, a fine wireframe grid.

The form performs the philosophy: the toggle in the top-right re-skins the entire page so you
can read it under either aspect. The palette is drawn from Justin's portrait, whose painted
forms already flow from green to graphite.

Built with [Astro](https://astro.build) (static), self-hosted fonts, and no runtime
dependencies. Publications live as Markdown files, so the list is easy to extend.

---

## Run it

```sh
npm install
npm run dev        # http://localhost:4321
```

```sh
npm run build      # outputs static site to ./dist/
npm run preview    # preview the production build
```

## It's deployed

Live at **https://msai-amin.github.io/justin-simpson/**, hosted on GitHub Pages.

Every push to `main` rebuilds and redeploys automatically via
`.github/workflows/deploy.yml` (the official `withastro/action`, pinned to Node 22).
So to publish a change: edit, commit, `git push` — done.

### Custom domain (optional, later)
To serve from a real domain (e.g. `justinsimpson.com`) instead of the `/justin-simpson/`
subpath: add the domain in the repo's **Settings → Pages**, then in `astro.config.mjs`
set `base: '/'` and `site: 'https://your-domain'`, and push. (With the subpath gone, the
`base`-prefixed asset links resolve at the root automatically.)

## Add a publication

Copy any file in `src/content/publications/` and edit the frontmatter:

```yaml
---
title: "Full title of the work"
year: 2026
venue: "Journal or Press, vol(issue), pages"
type: "article"        # article | chapter | review
coauthors: ["Name"]    # optional
url: "https://doi.org/..."  # optional
order: 1                # optional — higher sorts first within a year
summary: "One sentence in plain language."
---
```

## Change the portrait

Replace `src/assets/justin.jpg` (the build optimizes it automatically). The current image was
converted from the original with `sips`.
